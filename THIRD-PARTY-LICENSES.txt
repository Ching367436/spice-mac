3. Bundled versions (this release's sysroot)
--------------------------------------------

The native libraries come from a prebuilt UTM arm64 sysroot pinned per release.
Versions determined for the current build (the rest are recorded in, and
recoverable from, each release's `Frameworks/`):

  spice-client-glib (spice-gtk) .... 0.42
  glib / gobject / gio / gmodule ... 2.69.0
  gstreamer + base/good plugins .... 1.19.1
  libsoup .......................... 3.x (3.6.0)
  json-glib ........................ 1.0.x
  opus ............................. 1.3.x
  openssl .......................... 1.1.1w  (built by scripts/upgrade-openssl.sh)
  libusb, usbredir, phodav, pixman, libffi, libjpeg-turbo, libintl, libiconv
    ............................... as shipped in the pinned sysroot

The exact version of every component in a given release is fixed by the pinned,
SHA-256-checksummed sysroot tarball (`SPICEMAC_SYSROOT_URL`/`_SHA256`,
scripts/fetch-sysroot.sh) plus scripts/upgrade-openssl.sh.


4. LGPL-2.1 — notices and §6 written offer
------------------------------------------

The LGPL-2.1+ libraries above are used as dynamically-linked frameworks; the
GStreamer base/good plugins are statically linked into the SpiceMac executable.
A distributed SpiceMac.app therefore carries, under `Contents/Resources/Licenses/`,
a copy of the LGPL-2.1 license text and this notice (LGPL-2.1 §6 / §1).

Relinking with a modified library is supported and documented: SpiceMac is fully
open source (MIT). A recipient of a binary can rebuild it against their own
modified LGPL library — replace the relevant `.framework` under
`Contents/Frameworks/` (dynamic libraries) or rebuild from source with a modified
static archive (the statically-linked gst plugins) and re-run
`scripts/build-app.sh`, then re-sign. The complete build recipe is in the repo
(Package.swift, scripts/build-app.sh). This provides the §6(b)/§6(a) relink path.
